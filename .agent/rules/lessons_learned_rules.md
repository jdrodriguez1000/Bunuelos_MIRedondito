# REGLAS DE GESTIÓN DE LECCIONES APRENDIDAS (LESSONS_LEARNED_RULES)

## 1. Identificación y Control
*   **Código**: RL_LEARN_001
*   **Versión**: v1.0.0
*   **Estado**: Oficial
*   **Objetivo**: Asegurar que cada error o acierto se transforme en conocimiento accionable para Bunuelos SAS.

## 2. Criterios de Registro
*   **Obligatoriedad**: Se debe registrar al menos una lección aprendida al finalizar cada Fase del roadmap.
*   **Identificación Proactiva**: Si durante la ejecución de una tarea se identifica un "bloqueante" o un hallazgo técnico crítico, se debe registrar inmediatamente.
*   **Anatomía del Registro**:
    *   **Hecho**: Qué pasó.
    *   **Por Qué**: Causa raíz (sin buscar culpables, enfocado en procesos).
    *   **Acción**: Cómo evitar que pase de nuevo o cómo repetir el éxito.

## 3. Categorización
*   **TECHNICAL**: Relacionado con código, arquitectura, datos o modelos.
*   **BUSINESS**: Relacionado con lógica de negocio, reglas de dominio o stakeholders.
*   **PROCESS**: Relacionado con metodologías, comunicación o gestión del proyecto.

## 4. Revisión y Mejora Continua
*   Las lecciones aprendidas de la fase anterior deben ser leídas al iniciar la siguiente fase para ajustar los planes de implementación.
